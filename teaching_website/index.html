Quickstart

5-minute first success

Install Vectrune for your platform, then run your first app end to end.

Install Vectrune

🍺 macOS & Linux — Homebrew

brew tap davethomas11/homebrew-vectrune
brew install vectrune

After install, verify with vectrune --version.

🪟 Windows — winget

winget install Vectrune.Vectrune

Or download the latest .exe from the Releases page and add it to your PATH.

🦀 Build from source — any platform

cargo install --path .

Requires a Rust toolchain. Run from the cloned repository root.

1

Verify the install

vectrune --version
2

Run the example app

vectrune examples/app.rune

This starts the minimal REST app defined in examples/app.rune.

3

Hit the health route

curl http://127.0.0.1:3000/health

Expected response: OK

What you just learned: Vectrune apps use readable sections like @App and @Route, and you can get a working server without a lot of framework boilerplate.

Language tour

Read a Vectrune file at a glance

@App

Defines the app name, type, version, and optional startup behavior.

@Schema/Thing

Defines structure and validation rules for request or data shapes.

@Route/METHOD /path

Maps an HTTP route to a sequence of steps in a run: block.

Builtins in run:

Use builtins such as parse-json, validate, respond, and memory helpers.

#!RUNE

@App
name = Example API
type = REST
version = 1.0

@Route/GET /health
run:
    log "Health check"
    respond 200 "OK"

Guided example

Walk through a memory-backed API

The examples/memory_api.rune example teaches app startup, schema validation, and REST routes together.

1. Initialize app state

@App
name = Memory Example API
version = 1.0
type = REST
run:
    skaters = load-rune skateboarders.rune
    set-memory skaters

Load starter data at boot and keep it in memory for later routes.

2. Define a schema

@Schema/Skater
fields:
    name: String
    age: Integer
    style: String

Use schemas to validate the shape of incoming data before mutating state.

3. Read from memory

@Route/GET /Skaters
run:
    skaters = get-memory skaters
    return skaters

Routes can pull from runtime memory and return the current value directly.

4. Validate before write

@Route/PUT /Skaters
run:
    skaters = get-memory skaters
    new-skater = parse-json body
    validate new-skater #Skater
    append skaters.Skateboarder.record new-skater
    set-memory skaters
    return skaters

Parse the request, validate it against the schema, then update shared state.

Key builtins in this walkthrough: load-rune, set-memory, get-memory, parse-json, validate, append, and return.
