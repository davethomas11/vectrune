# Vectrune Teaching Website – User Stories

## Story 1: Interactive Playground MVP
- **As a** new user
- **I want** to edit and run .rune code in my browser
- **So that** I can quickly learn and experiment with Vectrune features.
- **Acceptance Criteria:**
  - Homepage links to playground.
  - Playground has code editor, run button, and output panel.
  - User can select and load example .rune files (e.g., memory_api.rune).
  - Output panel shows result or error from vectrune CLI backend.

## Story 2: Language Tour
- **As a** learner
- **I want** a step-by-step guide to Vectrune syntax and features
- **So that** I can understand how to build APIs and use builtins.
- **Acceptance Criteria:**
  - Tour covers App, Schema, Route, Builtins, Memory, Validation, etc.
  - Each section has code sample, explanation, and "Try it" button.

## Story 3: API Reference
- **As a** developer
- **I want** a searchable reference for builtins, schema, and config
- **So that** I can look up usage and examples quickly.
- **Acceptance Criteria:**
  - Table of builtins with name, description, usage, and example.
  - Schema types and validation rules.
  - Environment/configuration options.

## Story 4: Example Gallery
- **As a** user
- **I want** to browse and run real-world Vectrune examples
- **So that** I can see how to solve common API/data problems.
- **Acceptance Criteria:**
  - List of examples with description, code, and "Run" button.
  - Output panel shows expected result.
